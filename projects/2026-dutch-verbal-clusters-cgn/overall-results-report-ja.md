# オランダ語の動詞クラスター内語順をMaxEntモデルで分析してみました

## 1. このレポートの目的（Objective）

このレポートは、CGNを用いた動詞クラスター語順実験の最終結果を日本語でまとめたものである。

[本実験のGitHubリポジトリ（リンク）](https://github.com/juni-chen/research-repository/tree/817c4ee659d30a21c8f944ade53ef5399fe1ab12/projects/2026-dutch-verbal-clusters-cgn)

リサーチクエスチョンは以下の通りである：

```text
話し言葉のオランダ語・フラマン語の動詞クラスターにおいて、どのような要因が 1-2 / 2-1 語順と関係しているのか。
```

この分析では、オランダ語（Dutch）とフラマン語（Flemish）[^1]を同一言語の地域差としてではなく、別々の分析対象として扱う。

```text
Dutch   = CGNで Netherlands とラベル付けされたデータ
Flemish = CGNで Flanders とラベル付けされたデータ
```

## 2. はじめに（Introduction）

オランダ語は西ゲルマン語派に属する言語であり、英語やドイツ語と同じ系統に位置づけられる。

オランダ語の主節における基本語順は、定動詞が文の第二位置に現れる「V2語順」である。たとえば主節では、主語以外の要素が文頭に置かれた場合でも、定動詞は第二位置に現れる。一方、従属節では定動詞が節末に現れやすく、動詞的要素が文末付近にまとまって現れる。

オランダ語では、時制、相、態、モダリティなどを表すために、複数の動詞が組み合わされることが多い。たとえば、完了を表す *hebben*（元の意味は「持つ」）、*zijn*（元の意味は「〜である、〜にある、存在する」）、受動を表す *worden*「〜になる」、可能性や義務を表す法助動詞、さらに 、*laten*「〜させる（使役）」、*doen*「する（do-supportと呼ばれる文の真理条件に影響を持たない助動詞）」などの半助動詞的な動詞、*gaan*「行く」、*blijven*「留まる」、*komen*「来る」などの一般動詞を転用して動作の経過・変化を表現する助動詞的な何かが、別の動詞と結びつく。

この動詞同士（シャレじゃないよ）の結合は、動詞的要素が3つ、4つ、と増えていっても規則に従って問題なく産出することが可能でである。これは英語の助動詞構文と大きく異なるポイントですね（英語だと"could have done"のように接続できる要素とその順番にかなり制限がある）。このようなクラスタリングは、日本語の助詞類と異なり独立形態素同士の接続ではあるが、接続することで特殊な意味を持ち従属形態素のような働き方をすることも含め、かなり膠着語的な特徴を持っているように見える。動詞がクラスタリングする言語ってあんまり聞かないけど、統語論や文処理の研究テーマとして非常に面白いんじゃないかな。

ここで、主節の場合は、階層が一番上の動詞（定動詞 / finite verb）のみがV2位置に来て、その他の補語となる動詞は全て節末に移動する。従属節の場合は、V2というものはなく全ての動詞類が節末に来る。というのがまあ、ドイツ語のような「典型的にV2な言語」でV-final性の強い言語の文構造の説明として定番だが、オランダ語・フラマン語の場合は節末の動詞・動詞クラスタのさらに後ろに非動詞要素が外置（extrapose）される場合が非常に多いので、ドイツ語のような強いV-final性は仮定できない。この外置の有無は本実験でも一つの素性として考慮している。

実際にオランダ語で動詞クラスタリングが起こっている従属節の例は以下のようになる：

```text
dat     hij het boek    heeft   gelezen
that    he  the book    has     read

dat     hij het boek    gelezen heeft
that    he  the book    read    has
「彼がその本を読んだということ」
```

この2つは、どちらも助動詞 *heeft* と過去分詞 *gelezen* から成る二動詞クラスタを含んでいる。しかし、クラスタ内部の動詞類の順序が異なる。前者ではheadとなる助動詞が補語動詞に先行するのに対し、後者では逆にheadが後置される。このように、オランダ語の動詞クラスタ内語順は、揺れが許容されるという現象が知られている。ちなみにドイツ語では後者の語順のみ許容される。ドイツ語・オランダ語の従属節は基本的にV-finalであるとして分析できるが、ドイツ語ではクラスタ内語順も従属節全体と同様にhead-finalであり（少なくとも日本語を母語とする学習者にとって）自然に見えるのに対し、オランダ語は先の外置も含め、head-initialなのかhead-finalなのかよく分からない不思議な語順をしている。本実験では、この語順の差異を「1-2語順」と「2-1語順」として扱い、語順決定に関わる要素を統計的手法で調べる。

## 3. コーパスと抽出の概要

実験にはCorpus Gesproken Nederlands (CGN)のアノテーションデータを使用した。CGNのアノテーションアーカイブを、1行1トークン形式の正規化テーブルに変換しで利用した。

```text
data/processed/cgn_tokens.csv
```

変換後のコーパス規模は次の通りである。

| Language | Tokens |
| --- | ---: |
| Dutch | 6,388,126 |
| Flemish | 3,705,947 |
| Total | 10,094,073 |

私はさらに、ここから機械的に隣接する動詞類同士をクラスターとして抽出する作業を行なった。とりあえずの結果は以下の通りである：

| Language | Total Extracted Clusters | Known Adjacent Sequences | Target Subordinate Clusters | Excluded Without Complementizer |
| --- | ---: | ---: | ---: | ---: |
| Dutch | 83,583 | 49,149 | 25,017 | 24,132 |
| Flemish | 57,573 | 33,722 | 17,773 | 15,949 |
| Total | 141,156 | 82,871 | 42,790 | 40,081 |

自動的にクラスターを抽出するにあたり注意した点がある。

隣接する動詞類を機械的に抽出すると、主節のV2位置に来た定動詞と補語動詞がたまたま隣接する場合もクラスターとみなしてとってきてしまう。そのため本研究では、補文標識、たとえば *dat*「〜ということ」、*omdat*「〜なので」、*als*「もし／〜するとき」などによって導入される従属節に注目した。補文標識の後に現れ、節末付近でまとまる動詞列を分析対象とすることで、単なる偶然の隣接ではなく、統語的に意味のある動詞クラスタをより正確に取り出すことができている。実際には補文標識が現れないからといってそれが従属節でないということではないし、主節で3個以上動詞類が出現する場合の定動詞以外の2動詞が起こすクラスタリングなど主節にも動詞クラスターは存在しうるが、それらは機械的に抽出するのが難しかったので今回は無視した。それでも万単位という十分な数のデータを抽出できたと思う。

このような経緯で、実際のMaxEnt実験では、`1-2` / `2-1` の語順が判定でき、かつ前方に従属補文標識が検出されたクラスターのみを使用した[^2]。

## 4. 語順ラベルの付与

現在の変換済みCGNテーブルには、独立した `verb_rank` アノテーションがない。そのため、2動詞クラスターについては、定動詞を暫定的な主要部として扱い、次の規則で語順をラベル付けした。

\[
y(C) =
\begin{cases}
1\text{-}2 & \text{if the finite verb precedes the infinitive or parcitiple} \\
2\text{-}1 & \text{if the infinitive or parcitiple precedes the finite verb} \\
\text{unknown} & \text{otherwise}
\end{cases}
\]

例えば、次のクラスターは `1-2` とラベル付けされる。

```text
kon pinnen
```

一方、次のクラスターは `2-1` とラベル付けされる。

```text
gelezen heeft
```

## 5. 統計の概要（overview）

対象となる従属節内クラスターでは、DutchでもFlemishでも `1-2` が優勢だった。ただし、Flemishの方が微妙に `2-1` が多いようにも見える。

| Language | Order | Count | Proportion |
| --- | --- | ---: | ---: |
| Dutch | 1-2 | 18,521 | 0.7403 |
| Dutch | 2-1 | 6,496 | 0.2597 |
| Flemish | 1-2 | 12,196 | 0.6862 |
| Flemish | 2-1 | 5,577 | 0.3138 |

- Dutchでは、対象従属節クラスターの約 74.0% が `1-2` である。
- Flemishでは、対象従属節クラスターの約 68.6% が `1-2` である。
- 微妙な傾向ではあるが、FlemishはDutchよりも `2-1` が多く見える。

## 6. 主要部タイプの分類

主要部（head）タイプの分類は、何度か（私とCodexによる）議論の末に実態を反映した言語学的に妥当なものを考案して採用した。本実験で実際に使った分類は次の通りである：

| Head Type | Lemmas |
| --- | --- |
| `modal` | `kunnen`, `moeten`, `mogen`, `willen`, `zullen`, `hoeven` |
| `auxiliary` | `hebben`, `zijn`, `worden` |
| `semi_auxiliary` | `gaan`, `blijven`, `komen`, `laten`, `doen` |
| `other` | それ以外の識別可能な主要部 |
| `unknown` | 主要部が識別できない場合 |

特に困った点は、典型的な助動詞以外の扱いである。今回は `auxiliary` を助動詞として特に典型的である `hebben`, `zijn`, `worden` のみに限定し、`gaan`, `blijven`, `komen`, `laten`, `doen` のように元の動詞の意味がかなり残っており、動詞に話法性（modality）や文法的役割を付加するわけでないが他の動詞を補語としてとる用法のある動詞は `auxiliary` ではなく、`semi_auxiliary` として扱っている。

結論から言うと、主要部タイプは語順決定の要因として最も大きく中核的であることがわかっている。

| --- | --- | ---: | ---: | ---: | ---: |
| Dutch | auxiliary | 5,927 | 0.5171 | 5,534 | 0.4829 |
| Dutch | modal | 9,396 | 0.9815 | 177 | 0.0185 |

- `modal-head` は、DutchとFlemishの両方で非常に強い `1-2` 化傾向を示した。
- `semi_auxiliary-head` も、両方の言語で非常に強く `1-2` 化傾向を示した。

## 9. 補文標識から動詞クラスターまでの長さ

この分析における `sentence_length_type` は、コーパスからクラスターを抽出する際に指標として用いた補文標識から始めて、動詞クラスター本体に到達するまでの、補文標識と動詞クラスターそのものを除いた非動詞要素（non-verbal element）の数を使って算定している。

\[
\text{sentence\_length\_type}(C) =
\begin{cases}
\text{short} & \text{if a complementizer is found and } B(C) \leq 4 \\
\text{long} & \text{if a complementizer is found and } B(C) > 4 \text{ の場合} \\
\text{no\_complementizer} & \text{if no preceding complementizer is found}
\end{cases}
\]

ここで \(B(C)\) は、最も近い前方の従属補文標識、例えば `dat`, `omdat`, `als`, `of` などと、動詞クラスター本体との間にある非動詞要素の数である。

閾値として私（Codexのことではなく、実験を計画した人間）が設定した `4` は、極限まで短い従属節が主語NPと目的語NPを含むと仮定し、それぞれのNPを「冠詞 + 名詞」と近似する考えに基づいている。この程度なら十分「短い」とみなして、それ以上の、より多くの付加情報を含んだ節を「長い」とみなす。このように、本実験における節の「長さ」とは、閾値で切った「長い / 短い」の二項値である。

統計は以下のようになった：

| Language | Length Type | 1-2 Count | 1-2 Proportion | 2-1 Count | 2-1 Proportion |
| --- | --- | ---: | ---: | ---: | ---: |
| Dutch | long | 9,645 | 0.7594 | 3,055 | 0.2406 |
| Dutch | short | 8,876 | 0.7206 | 3,441 | 0.2794 |
| Flemish | long | 5,583 | 0.7016 | 2,375 | 0.2984 |
| Flemish | short | 6,613 | 0.6738 | 3,202 | 0.3262 |

それほど顕著ではないが、より長い節 `1-2` 語順が使われるという傾向が見られそうである。

## 10. 外置の有無（Existence of extraposition）

外置が存在するかどうかは、動詞クラスターの直後に続く要素が句読点であるかそれ以外の非動詞要素であるかの二項値で分類した。

\[
\text{cluster\_position}(C) =
\begin{cases}
\text{final} & \text{if no non-verbal material other than a punctuation follows} \\
\text{nonfinal} & \text{if material follows the cluster}
\end{cases}
\]

統計は以下のようになった：

| Language | Cluster Position | 1-2 Count | 1-2 Proportion | 2-1 Count | 2-1 Proportion |
| --- | --- | ---: | ---: | ---: | ---: |
| Dutch | final | 8,221 | 0.7667 | 2,501 | 0.2333 |
| Dutch | nonfinal | 10,300 | 0.7205 | 3,995 | 0.2795 |
| Flemish | final | 5,081 | 0.7111 | 2,064 | 0.2889 |
| Flemish | nonfinal | 7,115 | 0.6695 | 3,513 | 0.3305 |

これも弱い傾向だが、 `nonfinal` 、つまり外置を伴うクラスターでは `2-1` が多いように見える。

## 11. te不定詞（te-infinitive）の有無

本研究では、動詞クラスター内に te 不定詞が含まれるかどうかも特徴量として扱った。オランダ語では、不定詞が単独で現れる場合だけでなく、te + 不定形の形で現れる場合がある。たとえば、次のような例である。

```text
om          het boek    te  lezen
in  order   the book    to  read
「その本を読むために」
```

このte不定詞が別の動詞の補語として出現する場合もある：

```text
dat     hij het boek    probeert    te  lezen
that    he  the book    tries       to  read
「彼がその本を読もうとしていること」
```

本研究では、抽出された動詞クラスターについて、クラスター内またはその直前に te が検出される場合に `has_te = True` とし、それ以外を `has_te = False` とした。

## 12. 最大エントロピーモデル（MaxEnt model）の実装

本実験では、最大エントロピー（以下、MaxEntと呼ぶ）モデルを実装して、これらの要素がどのようにクラスター内語順決定に関わっているのかを複合的に分析している。

\[
P_{\Theta}(y=k \mid x)
=
\frac{\exp(\theta_k^\top \tilde{\phi}(x))}
{\sum_{m=1}^{K} \exp(\theta_m^\top \tilde{\phi}(x))}
\]

ここで、

- \(x\) は動詞クラスター
- \(y\) は語順ラベル
- \(k\) は可能な語順（ここでは `1-2` / `2-1` ）
- \(\tilde{\phi}(x)\) はバイアス項を含む素性ベクトル
- \(\theta_k\) はクラス \(k\) の重みベクトル

\(P_{\Theta}(y=k \mid x)\) は「動詞クラスターxの語順がk（ `1-2` or `2-1` ）である確率」で、それを素性ベクトル \(\tilde{\phi}(x)\) から導出できるような重みベクトル \(\theta\) を学習させ、重みベクトルの各成分を見ることでどの素性がどれほどの強さで語順決定に関わっているかを分析する。

まず、動詞クラスター \(x_i\) は素性ベクトル \(\tilde{\phi}(x_i)\) に変換される。

\[
\tilde{\phi}(x_i)
=
[1, \phi_1(x_i), \phi_2(x_i), \ldots, \phi_J(x_i)]
\]

先頭の \(1\) はバイアス項である。

素性ベクトルの中身は、先ほど設定した動詞クラスタの特徴ごとにそれぞれ、

```text
has_modal=True
has_te=True
cluster_position=final
sentence_length_type=long
```

のように値が入力される。

各語順ラベル \(k\)、つまり今回は `1-2` や `2-1` の2種類に対して、モデルはそれぞれ別の重みベクトルを持つ。

\[
\theta_k
=
[\theta_{0k}, \theta_{1k}, \theta_{2k}, \ldots, \theta_{Jk}]
\]

そして、あるクラスター \(x_i\) が語順 \(k\) になるスコアを次のように計算する。

\[
s_{ik}
=
\theta_k^\top \tilde{\phi}(x_i)
\]

このスコアをsoftmaxに通すことで、確率に変換する。

\[
P(y_i=k \mid x_i)
=
\frac{\exp(s_{ik})}
{\sum_{m=1}^{K}\exp(s_{im})}
\]

学習では、最初に \(\theta\) をほぼゼロに近いランダムな値で初期化し、各データについてモデルが予測した確率と実際の正解ラベルを比べる。

正解ラベルは、正解が `1-2` なら：

\[
t_i = [1, 0]
\]

正解が `2-1` なら：

\[
t_i = [0, 1]
\]

のようにone-hotベクトルで表現できる。

モデルの予測が：

\[
p_i = [P(1\text{-}2 \mid x_i), P(2\text{-}1 \mid x_i)]
\]

だとすると、誤差は：

\[
p_i - t_i
\]

になる。

この誤差を使って、損失関数を小さくする方向に \(\theta\) を少しずつ更新する。

\[
L(\Theta)
=
-\frac{1}{N}
\sum_{i=1}^{N}
\log P(y_i \mid x_i)
\]

重みの更新は勾配降下法で行っている。

まとめると、クラスターの語順を確率として与える重みベクトル \(\theta\) は、まず適当な初期値が設定される。その後、あるクラスターが本当は `1-2` なのに、モデルが `2-1` に高い確率を与えてしまった場合、次の更新で：

```text
そのクラスターに含まれていた素性の `1-2` 側の重みを上げる
そのクラスターに含まれていた素性の `2-1` 側の重みを下げる
```

ように調整される。逆に、本当は `2-1` なのに `1-2` と予測してしまった場合、その逆の調整が行われる。これをひたすら繰り返して（本実装では1500回）最終的な重みベクトルが学習された。

このモデルの最終的な正答率と対数損失は、オランダ語とフラマン語それぞれについて以下のようになる[^4]：

| Language | Target Subordinate Clusters | Accuracy | Log Loss |
| --- | ---: | ---: | ---: |
| Dutch | 25,017 | 0.7864 | 0.4008 |
| Flemish | 17,773 | 0.8118 | 0.3761 |

割と高い精度で推定できているように見える。

## 13. 係数の解釈

係数を解釈するには、各素性について `1-2` に対する重みと `2-1` に対する重みの差を比較すると分かりやすいと思われる。

\[
\Delta_j =
\theta_{j,1\text{-}2}
-
\theta_{j,2\text{-}1}
\]

\(\Delta_j > 0\) であれば、その素性は `2-1` 語順よりも `1-2` 語順を相対的により強く引き起こす原因となる。\(\Delta_j < 0\) であれば逆に、 `1-2` 語順よりも `2-1` 語順になる傾向を示す。

オランダ語（Dutch）で `1-2` 語順化傾向を示す素性は、以上の形で計算した強さの順番に以下の通りである：

| Feature | Delta Toward 1-2 |
| --- | ---: |
| `has_modal=True` | 1.3915 |
| `component=comp-k` | 1.0645 |
| `<bias>` | 0.8748 |
| `has_te=True` | 0.6791 |
| `has_semi_auxiliary=True` | 0.6423 |
| `has_auxiliary=False` | 0.5296 |
| `component=comp-o` | 0.2998 |

一方、フラマン語（Flemish）で `1-2` 語順化傾向を示す素性は以下の通りである：

| Feature | Delta Toward 1-2 |
| --- | ---: |
| `has_modal=True` | 1.4988 |
| `component=comp-o` | 0.9045 |
| `has_auxiliary=False` | 0.8700 |
| `<bias>` | 0.8529 |
| `has_semi_auxiliary=True` | 0.7089 |
| `has_te=True` | 0.6084 |
| `component=comp-k` | 0.2884 |

- `modal` は強い `1-2` 語順化傾向を持つ
- te不定詞を含むクラスターも `1-2` 語順化する傾向を持つ
- `semi_auxiliary` を含むクラスターも `modal` ほどではないが `1-2` 語順化傾向を持つ
- 逆に `auxiliary` が主要部である時はそれなりに強い `2-1` 語順化傾向がある。その強さは特にフラマン語において顕著であるように見える
- 節の長さ、外置の有無などの動詞クラスタ自体から見て外部に存在する要因は、動詞クラスタ内の主要部のタイプに比べるとはるかに小さい影響しかないと思われる

ただし、te不定詞に関しては、te不定詞を補語とする動詞が主に `auxiliary` よりも `semi-auxiliary` やその他の一般動詞に偏っている点が影響していると思われる。そのため、純粋にte不定詞の効果とは言えなそう[^3]。

## 14. 結論（Conclusion）

1. 一般傾向としてオランダ語でもフラマン語でも全体として `1-2` 語順が優勢でである。

2. フラマン語の方が多少強い `2-1` 語順化傾向がオランダ語に比べて見られるが、統計的な優位差検定などはしていないのでよくわからない。

3. 主要部タイプ（type of cluster heads）が語順を決める要素として最も大きい。特に、 `auxiliary` における `2-1` 語順化傾向と、その他のheadにおける `1-2` 語順化傾向が対照的である。

4. 一方で主要部タイプの違い以外の要因、つまりクラスタの外部環境に起因する要素は、本実験で考慮した範囲ではクラスタ内語順決定に大した影響を持たないと思われる。

## 15. 本実験の限界と今後の課題

1. CGNのアノテーションには独立した `verb_rank` がなく、本実験では2動詞クラスターの `1-2` / `2-1` ラベルを定動詞と補語動詞の順序から推定している。そのため、機械的に判定できない、例えば統語構造の複雑さや、文中の語の意味といった要素については検証できていない。

2. 節の長さの判定が従属補文標識の自動検出と機械的な単語数カウントに依存しているため、補文標識が省略されている場合、発話が非流暢な場合、またはタグ付けが想定と異なる場合、一部の従属節が見逃されていたり節の長さを正しく判定できていない可能性がある。

3. 現在のモデルは話者を明示的に考慮しておらず、実験に使われたデータが特定の話者やレジスターに偏っている可能性がある。
