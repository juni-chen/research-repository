# Conventions

## File Names

Use lowercase `kebab-case` names:

```text
evidentiality.md
japanese-case-particles.md
2026-06-15-reading-log.md
```

## Status Values

Use one of these values in metadata:

| Status | Meaning |
| --- | --- |
| `seed` | Quick initial capture. |
| `draft` | Usable but incomplete. |
| `review` | Needs checking against sources or data. |
| `stable` | Refined and reusable. |
| `archived` | Kept for record, no longer active. |

## Note Types

Common `type` values:

- `concept`
- `paper-note`
- `language-profile`
- `method`
- `example`
- `project`
- `annotation-guide`

## Linking

Prefer relative Markdown links:

```markdown
[Evidentiality](../notes/evidentiality.md)
```

When two notes are strongly connected, link both directions.

## Source Discipline

When a claim comes from a source, include a citation or short source note. Keep personal reflections separate from source summaries.
