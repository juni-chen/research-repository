# Repository Map

Use this file as the stable index for the repository.

## Core Areas

- `inbox/`: temporary capture.
- `notes/`: reusable knowledge.
- `references/`: sources and bibliographic records.
- `data/`: datasets, annotations, and links.
- `projects/`: active or completed research work.
- `analysis/`: shared computational work.
- `outputs/`: finished artifacts.
- `templates/`: reusable structures.

## Suggested Growth

Add subfolders only when a category has enough material to deserve one. For example:

```text
notes/concepts/
notes/languages/
notes/paper-notes/
data/raw/
data/processed/
data/annotations/
analysis/scripts/
analysis/notebooks/
```

The goal is a repository that remains easy to search, easy to cite, and easy to reuse.
